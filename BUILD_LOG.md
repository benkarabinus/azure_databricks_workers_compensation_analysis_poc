## Locked decisions

- **Use case:** California State Fund Lane 1 POC — RTW duration prediction (regression) + claims fraud investigation triage (classification). Frame fraud as *triage acceleration*, never "automated fraud detection."
- **Platform:** Azure Databricks **Serverless** only (no classic clusters). Medallion Bronze → Silver → Gold.
- **Unity Catalog:** single catalog `state_fund_poc`; schemas `bronze`, `silver`, `gold`, `config`, `security`, `ml`. Bronze tables prefixed `raw_`.
- **Storage:** dedicated ADLS Gen2 registered as a UC External Location (separate from metastore root).
- **Repo style:** CalmVault-like tutorial — `README.md` (landing), `DEPLOYMENT_GUIDE.md` (full walkthrough), `docs/` Jekyll Pages site (just-the-docs), session-per-folder.
- **Databricks App:** **Streamlit** (reads `gold.fraud_scores` only).
- **Synthetic data:** Python pandas/Faker, seeded; inject ~2–5% dirty patterns.
- **Docs rule:** every session step must cite verified Microsoft Learn / Databricks docs (no guessed URLs); keep `session-N/README.md`, `docs/session-N.md`, and `DEPLOYMENT_GUIDE.md` in sync.
- **Diagrams:** `.drawio` sources under `docs/diagrams/` + exported `.svg`/`.png`, Azure/Databricks icons; reference exported image in Markdown.

- **2026-06-24** — Session 0 teardown gotchas (operational): (1) A stale `~/.databrickscfg` DEFAULT profile (old workspace, different tenant) kept hijacking the databricks provider whenever it had no in-state workspace to target (during destroy/redeploy), causing cross-tenant `Invalid access token` / "external user" errors. Fix: retire it — `Move-Item $HOME/.databrickscfg $HOME/.databrickscfg.bak-<ts>`. This tutorial uses pure azure-cli auth, so the profile is pure liability. (2) `terraform destroy` of the RG failed (~10 min retry) because an **orphan workspace** existed in the RG but not in state; azurerm won't delete a non-empty RG by default. Fix: delete the orphan (`az resource delete --ids <workspace-id>`), then `terraform destroy`. NOTE: do NOT globally set `prevent_deletion_if_contains_resources = false` — it would force-delete a customer's existing RG when `create_resource_group = false`. (3) Deleting a workspace does NOT purge metastore-level UC objects (storage credential `ac-state-fund-poc`, external locations `sfpoc_landing`/`sfpoc_managed`) — they can orphan in the regional metastore and cause name conflicts on redeploy; clean via Catalog → External Data. State is now empty (clean slate).
